# Product

## Register

product

## Users

Operators who publish and manage short-drama content across desktop automation workflows. They use the app while preparing platform-specific configuration, starting automation runtimes, checking service state, and reviewing upload or publishing progress.

## Product Purpose

AutoDrama is a multi-platform desktop automation app for drama publishing workflows. The product should make platform automation feel controlled, observable, and modular, with each platform isolated in its own runtime, IPC, service, and page structure.

## Brand Personality

Precise, restrained, operational. The interface should feel like a focused desktop tool: quiet enough for repeated work, clear enough to expose runtime state, and confident enough to avoid decorative chrome.

## Anti-references

Avoid marketing-page composition inside the app, oversized decorative panels, platform logic leaking into shared UI infrastructure, and status displays that expose implementation details users do not need for the active workflow.

## Design Principles

- Keep platform boundaries visible in code and invisible in daily use.
- Prioritize operational clarity over decoration.
- Surface only the runtime state users can act on.
- Preserve a compact, familiar desktop-tool layout.
- Use shared UI components for structure, not platform behavior.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Keep interactions keyboard-accessible, avoid motion that does not communicate state, and ensure compact status indicators remain readable without relying on color alone.
